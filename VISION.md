# Retail Promo Agent

## Idea

Retail Promo Agent helps an eCommerce team discover slow-moving products and turn them into approved promotion campaigns.

Many promotion workflows start from guesswork, urgency, or manual analysis. Retail Promo Agent starts from product and sales signals.

Instead of starting with a blank campaign brief, the user starts with product and sales context. The agent looks at what is not moving, suggests promotion opportunities, explains its reasoning, and helps prepare the campaign material after human approval.

The product should feel like a focused workflow, not a generic chatbot. The user should always know what the agent is recommending, why it is recommending it, and what will happen next.

## Core Flow

1. The user logs in.
2. The user sees product and sales data.
3. The user asks the agent to find promotion opportunities.
4. The agent identifies slow-moving or underperforming products.
5. The agent explains why those products are good candidates.
6. The user selects an option and adjusts the promotion details.
7. The agent prepares a launch package.
8. The app saves the promotion, reasoning, approval, and generated assets.

## Launch Package

A promotion launch package can include:

- selected product or product group;
- discount suggestion;
- promotion window;
- campaign rationale;
- risks or caveats;
- customer-facing campaign copy;
- flyer or visual concept;
- operational checklist;
- approval history.

## Product Evolution

**V1: Product App**

A focused workflow for product data, promotion recommendations, approval, launch package, saved campaigns, and run history.

**V2: Realtime Voice**

Let users talk through campaign discovery, refinement, and approval.

**V3: Codex Skill**

Make the workflow available inside Codex as a reusable skill, using the app/API as the system of record.

## Boundaries

This product should avoid:

- becoming a generic chat assistant;
- pretending to replace the entire commerce platform;
- building a full checkout or order system;
- creating campaigns without approval;
- focusing only on flyer generation;
- adding too many unrelated features.
